# 待办事项（MVP：网页端战斗页）

## 项目目标
在网页端复刻《杀戮尖塔》的核心战斗体验，先做最小化模型：
- 角色：红裤衩（铁甲战士）
- 敌人：咔咔（Jaw Worm）
- 范围：只做战斗页面，不做卡组编辑/地图/事件/商店/遗物系统

## 优先级任务
1. 定义最小战斗规则（回合流程、能量、抽牌、弃牌、敌人行动）
2. 定义最小数据模型（玩家、敌人、卡牌、状态、战斗日志）
3. 实现战斗状态机（玩家回合 -> 敌人回合 -> 结算胜负）
4. 实现基础卡牌集（打击、防御、重击）
5. 实现咔咔意图与行为循环（攻击/叠甲/强化等最小集合）
6. 完成战斗页面 UI（手牌区、角色区、敌人区、意图区、日志区、结束回合按钮）
7. 做一轮可玩验证（可打完一场并正确结算）

## MVP 验收标准
- 可以在浏览器中完整进行一场 1v1 战斗
- 回合与数值结算稳定、可复现
- 胜利与失败状态正确触发
- 基础交互顺畅（出牌、结束回合、查看敌人意图）
